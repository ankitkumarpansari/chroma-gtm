## üìù Email Templates

### Email 1: Initial Outreach (Warm & Curious)

**Subject Lines (A/B Test):**
- A: "Quick question about your Chroma experience"
- B: "What made you choose Chroma?"
- C: "{Company} + Chroma - quick question"

**Body:**

```
Hi {FirstName},

I'm Ankit from the Chroma team. I noticed {Company} is using Chroma, and I wanted to reach out personally.

We're trying to understand what's actually working for teams like yours. Would you mind answering one quick question?

**What was the main reason you chose Chroma for {Company}?**

- [ ] Open source / self-hostable
- [ ] Developer experience / ease of use
- [ ] Performance / speed
- [ ] Cost (it's free)
- [ ] Specific feature: ___________
- [ ] Other: ___________

Just reply with a letter or a quick note. Takes 10 seconds.

If you have time for a 15-min call to share more, I'd love to hear your story. We're always looking to feature customers doing interesting things.

Thanks,
Ankit

P.S. - If Chroma isn't working well for you, I want to know that too. Honest feedback helps us improve.
```

**Characters**: ~750 | **Goal**: Get a response

---

### Email 2: Follow-up (Day 3, if no response)

## üìã Segmentation Strategy

### Segment by Company Type

| Segment | Email Tone | Focus |
|---------|------------|-------|
| **AI Startups** | Peer-to-peer, technical | Performance, scale |
| **Enterprise** | Professional, ROI-focused | Security, compliance |
| **Agencies/Consultants** | Partnership-oriented | Client use cases |
| **Individual Developers** | Casual, helpful | Learning, tutorials |

### Segment by Usage Signal

| Signal | Approach |
|--------|----------|
| **Active GitHub contributor** | Thank them, ask for feedback |
| **Cloud waitlist** | Offer early access |
| **High usage (if known)** | Offer case study opportunity |
| **New signup** | Welcome + onboarding help |

## üìù Response Handling

### If They Reply Positively
1. Thank them immediately
2. Send survey link
3. Offer to schedule a call
4. Add to "engaged ISV" list

### If They Share Feedback/Issues
1. Thank them for honesty
2. Log the feedback in product tracker
3. Offer to help resolve if possible
4. Follow up when issue is addressed

### If They're Not Using Chroma Anymore
1. Ask why they stopped
2. Log the reason
3. Thank them for their time
4. Keep on list for future re-engagement
